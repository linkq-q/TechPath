## 评级体系概览

基于对 B站 200+ 条 AI TA / 技术美术求职作品集视频的系统分析，建立以下相对评级标准。

## S 级特征（前 5%）

- **完整的 AI 工具链 Pipeline 展示**：从原始输入（参考图/草图）→ 中间过程（ControlNet/LoRA 参数）→ 最终输出，每步可复现
- **原创技术效果**：不只是跑现成模型，而是有自定义节点/自训 LoRA/自研 Shader 等原创成分
- **有工程文件或代码说明**：提供 ComfyUI 工作流 JSON / GitHub 仓库链接 / 关键参数说明文档
- **量化的效果对比数据**：如「优化后帧率从 45fps 提升至 72fps」「LoRA 训练后风格一致性提升 30%」

## A 级特征（前 5-20%）

- 有 2 个以上独立完整项目（不同技术方向）
- 技术栈覆盖广：AI 生成 + 渲染 + 工具/脚本 至少覆盖两个方向
- 有效果对比（Before / After），或有制作过程说明

## B 级特征（前 20-50%）

- 1 个完整项目 + 若干零散效果展示
- 技术方向单一但有一定深度
- 展示质量良好，视频剪辑清晰

## C 级特征（后 50%）

- 只有最终效果展示，无过程说明
- 技术标签模糊，无法判断实际掌握程度
- 多为课程跟做项目，缺乏独立实现

## 评分维度权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 技术深度 | 30% | 原理理解 + 自定义实现 |
| 项目完整度 | 25% | 有完整 pipeline + 可复现 |
| 技术广度 | 25% | 覆盖多个方向的技能点数量 |
| 展示质量 | 20% | 视频清晰度 + 说明详细程度 |

## 常见提升建议

- **C→B**：补充制作过程说明，哪怕只是文字描述
- **B→A**：增加一个不同技术方向的完整项目
- **A→S**：开源工作流或代码，加入量化对比数据
